# Clawbot for All 项目详细分析文档

## 一、项目概述

**Clawbot for All** 是一个多租户 OpenClaw 管理平台，为每个用户提供独立的 OpenClaw 实例，并预装微信插件。该项目基于 Node.js 22 开发，使用 Docker 容器化部署，支持 linux/amd64 和 linux/arm64 架构。

### 核心特性

- 邀请码注册机制
- 管理员初始化和强制首次改密
- 每用户独享一个 OpenClaw 实例
- 预装微信插件，支持扫码绑定
- 后台 runner 镜像预热，不阻塞服务启动
- 管理员后台查看实例状态、日志、模型配置
- 支持 CPU/内存限制，便于单机多实例部署
- 通过 GitHub Actions 自动发布镜像到 GHCR

## 二、项目架构

### 2.1 技术栈

- **运行时**: Node.js 22 (纯 ES Module)
- **容器化**: Docker + Docker Compose
- **基础镜像**: node:22-bookworm-slim
- **前端**: 原生 HTML/CSS/JavaScript (无框架)
- **数据存储**: 本地 JSON 文件 (data/db.json)
- **密码加密**: Node.js crypto (scrypt)
- **OpenClaw 版本**: 2026.6.1
- **微信插件**: @tencent-weixin/openclaw-weixin@2.4.4

### 2.2 系统角色

#### 1. 管理员 (Admin)
- 创建和管理邀请码
- 查看所有用户和实例状态
- 管理全局模型预设
- 查看服务器日志
- 手动刷新 runner 镜像
- 删除普通用户

#### 2. 普通用户 (User)
- 使用邀请码注册
- 创建一个专属 OpenClaw 实例
- 配置实例模型
- 启动/停止实例
- 绑定/解绑微信
- 通过代理访问 OpenClaw Dashboard

## 三、Docker 容器实现

### 3.1 应用容器 (clawbot-for-all)

**Dockerfile 分析** (`Dockerfile`):

```dockerfile
FROM node:22-bookworm-slim

# 安装 Docker CLI 和 tini (进程管理器)
RUN apt-get update && apt-get install -y docker.io tini ca-certificates

# 复制应用代码
COPY package.json ./
COPY public ./public
COPY src ./src
COPY containers ./containers

# 暴露端口 4300
EXPOSE 4300

# 使用 tini 作为 init 进程
ENTRYPOINT ["tini", "--"]
CMD ["node", "src/server.mjs"]
```

**关键点**:
- 安装 `docker.io` 包，以便容器内可以操作宿主机 Docker (通过挂载 docker.sock)
- 使用 `tini` 防止僵尸进程
- 无需 npm install，因为是纯 Node.js 标准库实现

### 3.2 Runner 容器 (openclaw-runner)

**Dockerfile 分析** (`containers/openclaw-runner/Dockerfile`):

```dockerfile
FROM node:22-bookworm-slim

ARG OPENCLAW_VERSION=2026.6.1
ARG OPENCLAW_WECHAT_PLUGIN_SPEC=@tencent-weixin/openclaw-weixin@2.4.4

# 安装依赖
RUN apt-get update && apt-get install -y git ca-certificates python3

# 全局安装 OpenClaw CLI
RUN npm install -g "openclaw@${OPENCLAW_VERSION}"

# 预装微信插件到临时目录
ENV OPENCLAW_PREINSTALLED_HOME=/opt/openclaw-preinstalled
RUN openclaw plugins install "${OPENCLAW_WECHAT_PLUGIN_SPEC}"
RUN cp -r <插件目录> "${OPENCLAW_PREINSTALLED_HOME}/.openclaw/extensions/openclaw-weixin/"

# 启动脚本
COPY containers/openclaw-runner/entrypoint.sh /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
```

**entrypoint.sh 逻辑**:

1. **初始化目录**: 创建 `${OPENCLAW_HOME}` 和 `/workspace`
2. **复制预装插件**: 从 `/opt/openclaw-preinstalled/.openclaw/` 复制到实际 home 目录
3. **创建符号链接**: 链接插件的 node_modules/openclaw 到全局安装位置
4. **生成默认配置**: 如果 `openclaw.json` 不存在，创建默认配置
5. **启动 OpenClaw Gateway**: `openclaw gateway --allow-unconfigured --bind lan --port 18789`

### 4.2 运行时管理 (`src/runtime.mjs`)

**职责**: Docker 容器操作、镜像管理、进程执行

**核心函数**:

#### 1. Runner 镜像管理

```javascript
// 后台预热镜像 (不阻塞启动)
warmRunnerImageInBackground()

// 拉取镜像 (带超时和重试)
pullRunnerImage({ trigger, force })

// 检查本地镜像
inspectRunnerImage()
```

**镜像状态机**:
- `idle` → `pulling` → `ready`
- `idle` → `missing` (本地不存在)
- `pulling` → `error` (拉取失败/超时)

#### 2. 容器生命周期

```javascript
// 启动实例容器
startInstance(projectRoot, paths, instance)
  - 确保镜像已拉取
  - 停止旧容器 (如果存在)
  - 解析挂载路径 (Docker in Docker 场景)
  - 启动容器并设置资源限制

// 停止实例容器
stopInstance(instance)
  - docker rm -f <container_name>

// 检查容器状态
inspectInstance(instance)
  - 返回 running/stopped 状态
```

**容器启动参数**:
```bash
docker run -d \
  --name clawbot-openclaw-<id> \
  --restart unless-stopped \
  --cpus "1.0" \
  --memory 1g \
  --network <shared_network> \
  -p <port>:18789 \
  -e OPENCLAW_HOME=/var/lib/openclaw \
  -e OPENCLAW_CONFIG_PATH=/var/lib/openclaw/openclaw.json \
  -v <host_home>:/var/lib/openclaw \
  -v <host_workspace>:/workspace \
  ghcr.io/zhangsen540445123/clawbot-openclaw-runner:latest
```

#### 3. 微信绑定流程

```javascript
startWechatBindJob(instance, handlers)
```

**流程**:
1. 执行 `openclaw channels login --channel openclaw-weixin`
2. 实时解析输出，提取二维码 (data URL 或 ASCII art)
3. 状态机: `starting` → `waiting_scan` → `scanned` → `connected`
4. 超时时间: 10 分钟 (可配置)

**输出解析**:
- Base64 二维码: 正则匹配 `data:image/[^;]+;base64,`
- ASCII 二维码: 匹配连续的 █▀▄▌▐▓▒░# 字符行
- 扫码状态: 关键词 `已扫码|scaned|scanned`
- 成功状态: 关键词 `连接成功|login confirmed`

#### 4. 交互式命令执行

```javascript
startInteractiveInstanceCommand(instance, command, options, handlers)
```

用于模型 OAuth 登录流程:
- 使用 `script` 包装 docker exec 以模拟 TTY
- 支持实时输入 (stdin)
- 支持输出回调和退出回调
- 超时机制

**安全特性**:

- scrypt(password, salt, keylen=64)

### 4.6 模型提供商 (`src/model-providers.mjs`)

**支持的模型提供商**:

   - API Key 认证
   - 支持自定义 Base URL

3. **Google Gemini API Key** (`google-api`)
   - API Key 认证
   - 模型: gemini-3.1-pro-preview

4. **Gemini CLI OAuth** (`google-gemini-cli`)
   - OAuth 认证
   - 通过 OpenClaw CLI 登录

5. **Anthropic API Key** (`anthropic-api`)
   - API Key 认证
   - 模型: claude-sonnet-4-6, claude-opus-4-8

6. **Anthropic setup-token** (`anthropic-setup-token`)
   - 外部 token 粘贴认证
   - 交互式登录流程

7. **Qwen OAuth** (`qwen-oauth`)
   - Device Code 认证
   - 阿里通义千问

**认证类型**:
- `api_key`: 直接填写 API Key
- `oauth_redirect_paste`: OAuth 后粘贴 redirect URL
- `device_code`: 设备码认证 (浏览器授权)

### 5.2 实例创建流程 (Provisioning)

**用户触发**: POST /api/instances

```
1. 验证用户权限 (requireUser)
   ↓
2. 检查用户实例数量 (限制 1 个)
   ↓
3. 解析模型配置:
   - 如果指定 presetId: 使用预设
   - 如果指定 providerId/modelId: 手动配置
   - 否则: 使用默认预设
   ↓
4. 分配端口 (从 19001 开始自动分配)
   ↓
5. 创建实例记录 (createInstanceRecord)
   ↓
6. 保存到数据库
   ↓
7. 启动后台 provisioning job
   ↓
8. 返回 202 Accepted
```

**Provisioning Job**:
```
1. 确保实例有默认模型
   ↓
2. 更新状态: "writing-config" (18%)
   ↓
3. 生成实例目录:
   - /app/data/instances/<id>/home/
   - /app/data/instances/<id>/workspace/
   - /app/data/instances/<id>/logs/
   ↓
4. 写入 openclaw.json 配置
   ↓
5. 更新状态: "starting-container" (56%)
   ↓
6. 拉起 Docker 容器 (startInstanceWithPortRecovery)
   ↓
7. 如果端口冲突: 自动重新分配端口并重试
   ↓
8. 更新状态: "ready" (100%)
   ↓
9. 完成
```

**宿主机执行**:
```bash
docker run -d \
  --name clawbot-openclaw-<instanceId> \
  --restart unless-stopped \
  --cpus "1.0" \
  --memory 1g \

**容器内执行** (entrypoint.sh):
```bash
#!/bin/sh
set -eu

# 1. 确保目录存在
mkdir -p "${OPENCLAW_HOME}" /workspace

# 2. 复制预装插件
if [ -d "${OPENCLAW_PREINSTALLED_HOME}/.openclaw" ]; then
  cp -R -n "${OPENCLAW_PREINSTALLED_HOME}/.openclaw/." "${OPENCLAW_HOME}/.openclaw/"
fi

# 3. 创建插件符号链接
PLUGIN_DIR="${OPENCLAW_HOME}/.openclaw/extensions/openclaw-weixin"
if [ -d "${PLUGIN_DIR}" ]; then
  ln -sfn /usr/local/lib/node_modules/openclaw "${PLUGIN_DIR}/node_modules/openclaw"
fi

# 4. 如果配置不存在，创建默认配置
if [ ! -f "${OPENCLAW_CONFIG_PATH}" ]; then
  cat > "${OPENCLAW_CONFIG_PATH}" <<'EOF'
{
  "gateway": { ... },
  "plugins": {
    "allow": ["openclaw-weixin"],
    "entries": { "openclaw-weixin": { "enabled": true } }
  }
}
EOF
fi

# 5. 启动 OpenClaw Gateway
exec openclaw gateway --allow-unconfigured --bind lan --port 18789
```

### 5.4 微信绑定流程

**用户触发**: POST /api/instances/:id/wechat-bind

```
1. 验证用户权限
   ↓
2. 确保实例 provisioning 完成
   ↓
3. 确保实例容器正在运行
   ↓
4. 更新微信状态: "starting"
   ↓
5. 启动微信绑定 job (startWechatBindJob)
   ↓
6. 在容器内执行:
   ```bash
   openclaw config set plugins.entries.openclaw-weixin.enabled true
   openclaw channels login --channel openclaw-weixin --verbose
   ```
   ↓
7. 实时解析输出:
   - 检测二维码 (data URL 或 ASCII)
   - 更新状态: "waiting_scan"
   - 前端轮询拉取最新状态
   ↓
8. 用户扫码:
   - 状态变为 "scanned"
   ↓
9. 微信确认登录:
   - 状态变为 "connected"
   - 解析账号信息
   ↓
10. 保存绑定记录到:
    /app/data/instances/<id>/home/.openclaw/openclaw-weixin/accounts.json
    ↓
11. 完成
```

**二维码提取逻辑**:
- **Data URL 模式**: 匹配 `data:image/png;base64,iVBORw0KGgo...`
- **ASCII 模式**: 提取连续的 █▀▄ 字符组成的二维码
- **链接提取**: 匹配 `https://` 开头的 URL

## 六、网络架构

### 6.1 代理模式

**场景 1: 应用容器和实例容器在同一 Docker 网络**
```
用户浏览器
  ↓ HTTP
应用容器 (clawbot-for-all:4300)
  ↓ HTTP (容器 IP:18789)
实例容器 (clawbot-openclaw-<id>)
```

**场景 2: 应用容器通过 host.docker.internal**
```
用户浏览器
  ↓ HTTP
应用容器 (clawbot-for-all:4300)
  ↓ HTTP (host.docker.internal:<port>)
宿主机
  ↓ 端口映射
实例容器
```

**场景 3: 应用直接运行在宿主机**
```
用户浏览器
  ↓ HTTP
Node.js (宿主机:4300)
  ↓ HTTP (127.0.0.1:<port>)
实例容器 (端口映射)
```

### 6.2 反向代理实现

**HTTP 代理** (`/proxy/:instanceId/*`):
- 验证用户权限
- 解析目标地址 (resolveInstanceProxyTarget)
- 注入 Authorization header (Bearer token)
- 转发请求和响应

**WebSocket 代理** (Upgrade):
- 拦截 HTTP Upgrade 请求
- TCP 层面直接转发
- 保持长连接

## 七、自动化运维

### 7.1 配置漂移修复

**场景**: 用户修改模型配置后，openclaw.json 可能与数据库不一致

**修复逻辑** (启动时自动执行):
```javascript
repairInstanceConfigDriftInBackground()
  - 对比 openclaw.json 和数据库配置
  - 如果不一致: 重新生成配置并重启容器
```

### 7.2 挂载漂移修复

**场景**: Docker in Docker 环境下，容器重启后挂载路径可能变化

**修复逻辑**:
```javascript
repairInstanceMountDriftInBackground()
  - 检查容器实际挂载点
  - 对比期望的挂载点
  - 如果不一致: 重启容器
```

### 7.3 空模型修复

**场景**: 旧数据库中实例没有 model 字段

**修复逻辑**:
```javascript
repairInstancesMissingModelInBackground()
  - 查找 model 为空的实例
  - 自动分配默认模型
  - 更新配置并重启容器
```

### 8.1 应用容器环境变量

```bash
# 服务器配置
HOST=0.0.0.0                    # 监听地址
PORT=4300                        # 监听端口
SESSION_TTL_DAYS=14              # Session 有效期 (天)

# 管理员账号 (首次启动自动创建)
ADMIN_EMAIL=admin@example.com
ADMIN_NAME=平台管理员
ADMIN_PASSWORD=ChangeMe123!

# Runner 镜像配置
OPENCLAW_RUNNER_IMAGE=ghcr.io/zhangsen540445123/clawbot-openclaw-runner:latest
OPENCLAW_RUNNER_PULL_TIMEOUT_MS=600000   # 拉取超时 (毫秒)
OPENCLAW_RUNNER_CPUS=1.0                 # 每个实例 CPU 限制
OPENCLAW_RUNNER_MEMORY=1g                # 每个实例内存限制

# 微信绑定超时
OPENCLAW_WECHAT_BIND_TIMEOUT_MS=600000

# Docker 网络 (可选，自动检测)
CLAWBOT_DOCKER_NETWORK=clawbot_default

## 九、部署方式

### 9.1 Docker Compose 部署

**compose.yaml**:
```yaml
services:
  clawbot:
    image: ghcr.io/zhangsen540445123/clawbot-for-all:latest
    container_name: clawbot-for-all
    restart: unless-stopped
    ports:
      - "4300:4300"

## 十一、安全机制

- Session 管理: Cookie-based, HttpOnly
- 密码存储: scrypt + 随机 salt
- 实例隔离: 独立容器 + 独立文件系统
- Token 鉴权: 每实例独立 gatewayToken
- 代理安全: 权限验证 + Cookie 隔离
